# 埃希斯特利亚状态栏 0.5.0 交互结构文档

## 设计目标

0.5.0 状态栏采用移动端竖屏优先设计。它不是一个独立 App 首页，也不需要铺满整个手机屏幕，而是衔接在正文后的阅读辅助组件：玩家读完剧情后继续下滑，即可看到当前 MVU 状态摘要，并按需展开详情。

视觉基准采用「掌心控制中心风」：深色石墨青主面板、冷蓝状态强调、暖白详情抽屉、盾徽式身份感、圆角磁贴、进度条、手风琴、底部抽屉和弹窗。默认态负责好看和速读，展开态负责常用变量，详情态负责完整变量。

## 变量边界

状态栏只能展示 `schema.ts` 中真实存在的 MVU 变量，不添加伪功能入口。

禁止出现的伪入口或伪变量：

- `上路许可`
- `异常警报`
- `档案`
- `位置追踪`
- `通行权限`
- `通知`
- `菜单`
- `查看记录`

允许出现的是变量分组、详情入口和交互控件，例如：

- `当前场景`
- `待办事项`
- `骑士`
- `女骑`
- `关系背景`
- `外貌记录`
- `身材记录`
- `装备`
- `机动能力`
- `查看详情`
- `展开`
- `收起`
- `返回状态卡`

## 信息层级

### 一级：默认状态卡

默认状态卡是正文后方最先出现的状态栏主体，建议高度控制在手机竖屏半屏到三分之二屏左右。内容多时允许继续向下自然延展，但默认态不追求完整展示所有长文本。

默认展示：

- `主角.姓名`
- `主角.年龄`
- `主角.san值`
- `当前场景.时间`
- `当前场景.地点`
- `当前场景.天气.概况`
- `当前场景.天气.气温`
- `当前场景.天气.风`
- `待办事项.主要` 摘要
- `骑士` 记录列表入口
- `女骑` 记录列表入口

默认不展示完整长文本：

- `当前场景.环境`
- `待办事项.主要` 全文
- `待办事项.次要` 全文
- `骑士.*.外貌`
- `骑士.*.性格`
- `骑士.*.当前心理`
- `骑士.*.当前动作`
- `女骑.*.关系背景`
- `女骑.*.外貌`
- `女骑.*.身材`
- `女骑.*.装备.*.描述`

`骑士` 与 `女骑` 均采用手风琴组件。每条记录默认收起，点击后展开当前人物摘要。

骑士展开态展示：

- `身份`
- `好感度`
- `态度` 摘要
- `当前心理` 摘要
- `当前动作` 摘要
- `查看完整骑士详情` 入口

女骑展开态展示：

- `年龄`
- `身份`
- `所属骑士`
- `所属关系`
- `好感度`
- `体力`
- `态度` 摘要
- `当前心理` 摘要
- `当前动作` 摘要
- `查看完整女骑详情` 入口
- 五个二级详情入口：
  - `关系背景`
  - `外貌记录`
  - `身材记录`
  - `装备`
  - `机动能力`

### 三级：详情抽屉与弹窗

详情层用于展示完整变量。优先使用底部抽屉，因为它更符合移动端阅读场景；装备可使用弹窗，因为其结构更像离散物品列表。

详情组件类型：

- 底部抽屉：场景与待办、骑士详情、女骑详情、关系背景、外貌记录、身材记录、机动能力。
- 弹窗：装备详情。

### 12 SAN 变化状态

用途：定义 `主角.san值` 的视觉状态，而不是新增变量。

组件类型：SAN 组件状态规则。

状态规则：

- `80-100`：正常，冷蓝色。
- `50-79`：警戒，琥珀色。
- `1-49`：危险，深红色。
- `0`：崩溃边界，深红加强并可添加轻微脉冲。

显示规则：

- 文案使用 `SAN {value}`。
- 不显示 `异常警报` 等伪入口。
- 可显示状态词：`稳定`、`警戒`、`危险`。
- SAN 变化只表达当前状态，不主动解释原因。

交互：

- 点击 SAN 区域可打开轻量说明浮层。
- 浮层只说明当前数值和状态，不编造额外剧情。

## 交互状态模型

建议使用一个简单的本地 UI 状态对象管理交互：

```ts
type OpenPanel =
  | null
  | { type: 'sceneTodo' }
  | { type: 'knightDetail'; name: string }
  | { type: 'femaleRiderDetail'; name: string }
  | { type: 'femaleRiderRelation'; name: string }
  | { type: 'femaleRiderAppearance'; name: string }
  | { type: 'femaleRiderBody'; name: string }
  | { type: 'equipment'; name: string }
  | { type: 'mobility'; name: string }
  | { type: 'sanState' };

type ExpandedCharacter =
  | null
  | { kind: 'knight'; name: string }
  | { kind: 'femaleRider'; name: string };
```

推荐规则：

- `ExpandedCharacter` 控制手风琴。
- `OpenPanel` 控制详情抽屉、弹窗和 SAN 浮层。
- 打开详情层时不必清除手风琴状态，关闭后回到原上下文。
- 同一时刻只允许一个详情层打开。

## 文本展示规则

长文本默认采用摘要 + 展开详情：

- 默认态：最多 1 行到 2 行，超出省略。
- 手风琴展开态：最多 2 行到 3 行。
- 详情抽屉：显示完整文本。

摘要不应改写设定含义，只做截断或自然省略。若需要自动摘要，必须保留原文可展开查看。

## 响应式与移动端规则

- 状态栏最大宽度跟随酒馆消息区域，手机端占满可用宽度。
- 默认态不强制满屏。
- 详情抽屉最大高度可接近视口高度的 80%，内部允许滚动。
- 弹窗宽度不超过屏幕宽度的 92%。
- 所有可点击区域高度建议不低于 44px。
- 手风琴行、入口卡、关闭按钮需要明显可点击反馈。

## 视觉规则

- 主色：深石墨青、雾黑、冷蓝。
- 详情面：暖白、浅灰、低饱和蓝灰。
- 女骑强调： muted crimson，不使用刺眼粉色。
- 骑士强调：冷蓝或钢蓝。
- SAN 正常：冷蓝。
- SAN 警戒：琥珀。
- SAN 危险：深红。

避免：

- 大面积紫色渐变。
- 伪玻璃拟态。
- 过度装饰的卡片堆叠。
- 表格化清单感。
- 桌面仪表盘布局。
