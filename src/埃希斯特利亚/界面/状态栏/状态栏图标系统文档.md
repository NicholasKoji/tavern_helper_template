# 埃希斯特利亚状态栏 0.5.0 图标系统文档

## 设计目标

状态栏图标不使用通用前端图标库作为主要视觉来源。0.5.0 的图标需要服务于「掌心控制中心风」：冷色终端、社会管理局档案感、骑士制度符号、移动端高识别度。

图标的作用不是装饰，而是帮助玩家快速区分变量分组、交互入口和状态强弱。所有图标必须围绕 `schema.ts` 中真实存在的变量或通用交互控件设计，不新增伪变量入口。

## 总体原则

- 核心图标采用定制 SVG。
- 通用交互图标采用同风格简化 SVG。
- 不直接使用 lucide、heroicons、fontawesome 等图标库作为最终图标。
- 图标必须可被统一换色、缩放和添加发光效果。
- 图标风格保持克制，不做复杂插画。
- 图标优先表达变量分组，不表达不存在的功能。

## 禁止图标

以下图标不应出现在状态栏中，因为它们会暗示不存在的变量或伪功能：

- `上路许可`
- `异常警报`
- `档案`
- `位置追踪`
- `通行权限`
- `通知`
- `菜单`
- `摄像头监控`
- `地图导航`
- `权限审核`
- `任务系统`

如果需要表达真实变量，应改用对应变量图标。例如 `当前场景` 使用场景图标，`待办事项` 使用待办图标，`装备` 使用装备图标。

## 图标语法

### 基础形态

图标以线性 SVG 为主，允许局部半透明填充。

推荐规则：

- 画布：`24x24`。
- 默认线宽：`1.8`。
- 大尺寸徽章线宽：`2` 到 `2.2`。
- 端点：`round`。
- 连接：`round`。
- 主图形尽量使用几何轮廓。
- 复杂图标控制在 3 到 5 个主要路径内。

### 视觉特征

图标应具有轻微「制度终端」感：

- 盾形、刻线、分割线适合身份类图标。
- 环形、指针、刻度适合状态类图标。
- 矩形、短横线、标签片适合记录类图标。
- 轮组、轨迹线、速度刻度适合机动类图标。
- 心电线、脉冲线适合 SAN 和体力类图标。

避免：

- 过度圆润的应用图标感。
- 卡通表情。
- 复杂写实细节。
- 与变量无关的武器、警报器、定位针、铃铛。

## 尺寸系统

状态栏建议使用三档尺寸：

- `sm`：`18px`，用于文本旁边的辅助图标。
- `md`：`24px`，用于普通入口和按钮。
- `lg`：`36px`，用于人物行、主指标、分组标题。
- `badge`：`48px` 到 `56px`，用于主角、骑士、女骑的身份徽章。

点击区域不能只等于图标本身。移动端按钮点击热区建议不低于 `44px`。

## 色彩系统

图标本体通过 CSS 变量控制，不在 SVG 内写死颜色。

推荐 token：

```css
--icon-main: #e8edf0;
--icon-muted: #91a4ad;
--icon-blue: #67b7f7;
--icon-cyan: #75d8e6;
--icon-crimson: #d66b70;
--icon-amber: #d9a84d;
--icon-danger: #b8474f;
--icon-dark: #20313a;
```

状态规则：

- 默认：`--icon-main`
- 弱信息：`--icon-muted`
- 骑士：`--icon-blue`
- 女骑：`--icon-crimson`
- 机动能力：`--icon-cyan`
- SAN 稳定：`--icon-blue`
- SAN 警戒：`--icon-amber`
- SAN 危险：`--icon-danger`
- 暖白抽屉中的图标：`--icon-dark`

发光效果只用于核心状态，不应所有图标都发光。

## 图标清单

### 核心身份图标

#### `crest`

用途：状态栏标题、主徽章、系统身份感。

视觉：盾形外框，中轴竖线，内部短横刻线。它是整个状态栏的视觉锚点。

使用位置：

- 默认状态卡标题。
- 主角摘要。
- 人物徽章背景。

#### `protagonist`

用途：主角信息。

视觉：盾徽基础上加入小型人形刻印或中心竖标。避免做成普通用户头像。

对应变量：

- `主角.姓名`
- `主角.年龄`

#### `knight`

用途：骑士记录。

视觉：盾形加竖刃感中轴，强调制度身份和管理局秩序。

对应变量：

- `骑士`
- `骑士.*.身份`
- `骑士.*.好感度`

#### `female-rider`

用途：女骑记录。

视觉：盾形更窄，底部可加入轻微轮组弧线，强调骑乘单位身份。不要画成性感人体图标。

对应变量：

- `女骑`
- `女骑.*.身份`
- `女骑.*.所属骑士`
- `女骑.*.所属关系`

### 场景与状态图标

#### `time`

用途：当前时间。

视觉：简化圆形表盘，单根指针和一条刻度。

对应变量：

- `当前场景.时间`

#### `location`

用途：当前地点。

视觉：三层区域节点或建筑定位牌。避免使用通用地图 pin。

对应变量：

- `当前场景.地点`

#### `weather`

用途：天气摘要。

视觉：低矮云层加短风线，适合阴天和海风。

对应变量：

- `当前场景.天气`

#### `environment`

用途：环境详情。

视觉：房间框线、角标和一条空气纹理线。

对应变量：

- `当前场景.环境`

#### `todo`

用途：待办事项。

视觉：短记录板与勾选刻线。

对应变量：

- `待办事项.主要`
- `待办事项.次要`

### 指标图标

#### `san`

用途：SAN 状态。

视觉：环形读数或脑波脉冲线。不要使用警报三角形作为默认样式。

对应变量：

- `主角.san值`

状态：

- `stable`
- `warning`
- `danger`
- `collapse`

#### `stamina`

用途：女骑体力。

视觉：心电线和能量槽结合。

对应变量：

- `女骑.*.体力`

#### `favor`

用途：好感度。

视觉：小型菱形刻度或亲密度节点，不使用爱心作为主图标，避免风格跳脱。

对应变量：

- `骑士.*.好感度`
- `女骑.*.好感度`

### 女骑详情图标

#### `relation`

用途：关系背景。

视觉：两枚小盾或两个节点之间的短链。

对应变量：

- `女骑.*.所属骑士`
- `女骑.*.所属关系`
- `女骑.*.关系背景`

#### `appearance`

用途：外貌记录。

视觉：档案框内的人像轮廓线，但不使用真实肖像。

对应变量：

- `女骑.*.外貌`

#### `body-record`

用途：身材记录。

视觉：测量刻度、曲线轮廓和记录标签。保持暧昧但不露骨。

对应变量：

- `女骑.*.身材`

#### `equipment`

用途：装备详情。

视觉：模块化护具、扣带或装配槽。

对应变量：

- `女骑.*.装备`

#### `mobility`

用途：机动能力。

视觉：轮组、速度弧线、轨迹刻度。

对应变量：

- `女骑.*.机动能力`

### 交互图标

#### `chevron`

用途：展开、进入详情。

视觉：单线箭头。

#### `close`

用途：关闭抽屉或弹窗。

视觉：两条交叉线。

#### `back`

用途：返回状态卡或返回女骑详情。

视觉：左向短箭头。

#### `expand`

用途：展开小分组。

视觉：向下箭头或短折线。

#### `collapse`

用途：收起小分组。

视觉：向上箭头或短折线。

## 实现方式

建议做一个统一的 `StatusIcon` 组件，而不是在各个组件中散落 SVG。

推荐 API：

```ts
type StatusIconName =
  | 'crest'
  | 'protagonist'
  | 'knight'
  | 'female-rider'
  | 'time'
  | 'location'
  | 'weather'
  | 'environment'
  | 'todo'
  | 'san'
  | 'stamina'
  | 'favor'
  | 'relation'
  | 'appearance'
  | 'body-record'
  | 'equipment'
  | 'mobility'
  | 'chevron'
  | 'close'
  | 'back'
  | 'expand'
  | 'collapse';

type StatusIconTone =
  | 'main'
  | 'muted'
  | 'blue'
  | 'cyan'
  | 'crimson'
  | 'amber'
  | 'danger'
  | 'dark';

type StatusIconSize = 'sm' | 'md' | 'lg' | 'badge';
```

推荐用法：

```vue
<StatusIcon name="san" tone="blue" size="lg" />
<StatusIcon name="female-rider" tone="crimson" size="badge" />
<StatusIcon name="mobility" tone="cyan" size="md" />
```

组件职责：

- 根据 `name` 渲染对应 SVG。
- 根据 `tone` 设置 CSS 变量。
- 根据 `size` 设置统一尺寸。
- 支持 `active`、`dimmed`、`glow` 状态。
- 默认 `aria-hidden="true"`，若图标按钮无文字，则按钮必须提供 `aria-label`。

## SVG 编写规范

每个图标都应符合以下结构：

```vue
<svg
  viewBox="0 0 24 24"
  fill="none"
  stroke="currentColor"
  stroke-width="1.8"
  stroke-linecap="round"
  stroke-linejoin="round"
>
  <!-- paths -->
</svg>
```

允许局部填充：

```vue
<path class="icon-soft-fill" d="..." />
```

对应 CSS：

```css
.icon-soft-fill {
  fill: currentColor;
  opacity: 0.16;
  stroke: none;
}
```

不允许：

- 在 SVG path 上写死 `#67b7f7` 等颜色。
- 每个图标单独设置不同线宽。
- 使用外链图片作为图标。
- 使用 emoji 代替图标。

- `todo`
- `knight`
